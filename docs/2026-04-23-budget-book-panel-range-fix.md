# 预算书拖拽面板范围修正说明

## 本次修正内容

- 修正预算书工作区中两层上下拖拽面板的可拖拽范围：
  - 外层主树表 / 下方详情区分隔条
  - `取费程序 -> 显示费率` 内层树表 / 费率表分隔条
- 两层分隔条现在都支持：
  - 向上拖到只保留上方表格表头区域
  - 向下拖到下方面板完全不可见
  - 下方面板隐藏后，仍可通过分隔条再次向上拖出

## 本次实现调整

- 将外层详情区高度最小值从固定值调整为 `0`。
- 将内层费率表高度最小值从固定值调整为 `0`。
- 将两层上半区的保底剩余高度统一收敛到 `56`，用于保留表头和基础交互区。
- 将两层 `grid-template-rows` 的上半区从固定最小高度改为 `minmax(0px, 1fr)`，去掉原先限制拖拽范围的硬约束。

## 原因说明

- 此前不仅拖拽 hook 做了最小高度限制，布局层的 `grid-template-rows` 也额外写死了上半区最小高度。
- 两层限制叠加后，分隔条无法继续向上或向下移动到目标位置，所以实际拖拽范围比预期小很多。

## 影响文件

- `src/views/cost/project-management/budget-book.vue`
  - 调整外层详情区拖拽范围
  - 调整内层费率区拖拽范围
  - 调整上下两层 grid 布局最小约束
