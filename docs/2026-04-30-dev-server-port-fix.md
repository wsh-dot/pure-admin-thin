# 开发服务端口冲突修复说明

## 新增内容

- 将本地开发服务默认端口从 `8848` 调整为 `8850`，避免与当前机器上已监听 `8848` 的 `java.exe` 进程冲突。
- 同步更新环境变量配置和代码中的兜底默认端口，确保删除或漏读环境变量时也不会回退到冲突端口。

## 实现功能

- 执行 `pnpm dev` 时，Vite 默认监听 `8850`。
- 如需临时指定其他端口，仍可通过命令行参数覆盖，例如 `pnpm dev -- --port 8860`。

## 对应文件

- `.env`
- `.env.development`
- `build/utils.ts`

## 问题原因

- 原默认端口 `8848` 已被本机 `java.exe` 进程占用。
- Vite 再次监听 `0.0.0.0:8848` 时触发 `listen EACCES: permission denied 0.0.0.0:8848`。
