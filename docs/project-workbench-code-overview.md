# 项目管理工作台代码说明

## 1. 这份文档是干什么的

这份文档是给刚接手这个页面的同学看的。

目标很简单：

- 告诉你这次为了实现“项目管理工作台”新增了哪些源码文件
- 每个文件是干什么的
- 每个文件解决了什么问题
- 这些文件之间是怎么配合起来工作的

这份文档只统计和“项目管理工作台”功能直接相关的源码文件。
像 `.tmp-*` 这类临时文件、原型截图 HTML、日志文件，不算在这里面。

---

## 2. 先用一句大白话讲整体结构

现在这个功能可以简单理解成：

1. 先通过路由把“项目管理工作台”这个页面挂到系统菜单里
2. 进入页面后，由一个总控页面 `index.vue` 负责切换不同工作区
3. 不同工作区分别有自己的子组件
   - 项目管理主界面
   - 项目工料机汇总
   - 项目报表
4. 页面里用到的数据，大部分先放在 mock 文件里
5. 表格统一使用 `vxe-table`
6. 右侧属性面板的拖拽收起，是通过一个独立的 hook 来处理的

如果把它想成搭积木：

- `路由文件` 负责“把页面挂出来”
- `index.vue` 负责“总调度”
- `components` 目录里的文件负责“具体长什么样、怎么交互”
- `mock` 文件负责“先喂假数据”
- `types.ts` 负责“约定每块数据长什么样”
- `hooks` 负责“提取公共交互逻辑”

## 4. 每个文件展开讲，小白也能看懂

### 4.1 `src/router/modules/projectWorkbench.ts`

这个文件的作用只有一个：把页面接进系统菜单和路由。

可以把它理解成“商场导览图”。  
如果没有这个文件，页面代码写得再好，系统也不知道去哪里打开它。

它做了两件事：

- 定义了顶级菜单 `在线造价`
- 定义了子页面 `项目管理工作台`

顺手还加了一个很实用的设置：

- `hideFooter: true`

这个意思就是：进入工作台页面时，把布局底部的页脚隐藏掉，这样页面更像桌面软件，而不是普通网站内容页。

---

### 4.2 `src/plugins/vxeTable.ts`

这个文件很小，但很关键。

因为现在页面里大量地方都用了 `vxe-table`：

- 主树表
- 右侧属性树表
- 工料机汇总表

如果不先把 `vxe-table` 注册进 Vue，全页面这些表格都跑不起来。

所以这个文件可以理解成：

- “先把表格引擎装到应用里”

---

### 4.3 `src/views/cost/project-management/index.vue`

这是整个功能最核心的文件。

如果把整个页面比作一个办公室，这个文件就是“总控台”。

它主要负责下面这些事情：

#### 1. 页面总布局

它决定什么时候显示哪一套工作区：

- 默认显示 `项目管理`
- 切到 `项目工料机汇总` 时，显示另一套左树右表界面
- 切到 `项目报表` 时，再显示报表专用界面

也就是说：

- 真正的“页面切换逻辑”在这里

#### 2. 顶部工具栏切换

不同主页签，对应不同工具栏。

比如：

- `项目管理` 有新建、打开、导入、导出、保存这些按钮
- `项目工料机汇总` 有保存、导出excel、批量刷价、汇总这些按钮
- `项目报表` 有导入模板、预览、打印、导出PDF、设置报表这些按钮

这些切换都是它控制的。

#### 3. 项目管理主界面数据和选中逻辑

它负责：

- 当前选中了哪个阶段
- 当前表格选中了哪一行
- 右侧属性面板展示哪个节点的信息
- 审核意见展示哪个节点的说明

#### 4. 新建建设项目弹窗

它实现了：

- 点击“新建”
- 选择“新建建设项目”
- 弹出输入编号和名称的对话框
- 点确定后，在主树表里新增一条父级项目节点

#### 5. 新建造价文件弹窗

它实现了：

- 点击“新建”
- 选择“新建造价文件”
- 弹出“创建造价文件”对话框
- 支持切换 `石油 / 陕西 / 甘肃 / 宁夏 / 公路 / 电力 / 新能源 / 通信 / 其他`
- 不同分类会切换不同的：
  - 计价依据
  - 主定额
  - 项目模板
  - 工程所在地
- 点确定后，会在当前父节点下面插入一个新的造价文件子节点

#### 6. 项目报表进入逻辑

它实现了两个不同入口：

- 如果当前选中的是造价文件叶子节点：
  - 先弹确认框
  - 确认是否加载该造价文件报表
- 如果当前选中的是父节点：
  - 直接进入项目报表页

#### 7. 右侧属性面板

它把右侧属性面板接进来，并控制：

- 显示/隐藏
- 当前激活页签是 `基本信息` 还是 `审核意见`
- 宽度变化
- 收起/展开

可以理解成：

- 页面的大部分“业务交互总控”都在这个文件里

### 4.6 `src/views/cost/project-management/projectReportMock.ts`

这个文件只服务于 `项目报表` 页面。

可以把它理解成：

- “报表页的剧本”

报表页比普通表格复杂很多，因为它不是简单显示一张表，而是要：

- 显示左侧报表树
- 点击不同叶子节点时切换不同报表模板
- 有些报表还有多页

这个文件就把这些内容提前组织好了。

它主要做了下面几件事：

#### 1. 统一生成表格报表页

比如 `createTablePage()` 这种方法，就是为了少写重复代码。

它能快速生成：

- 标题
- 页码
- 表头
- 表格行
- 页脚

#### 2. 生成“取费计算表”多页报表

比如：

- 一般安装工程取费计算表
- 人工土石方工程取费计算表
- 机械土石方工程取费计算表

这些报表结构很像，所以用了公共生成逻辑。

#### 3. 生成“审批表”报表

比如：

- 工程预算审查审批表(审预表一)

这种报表有自己的布局，所以也单独做了生成逻辑。

#### 4. 生成左侧报表树

包括：

- 编制报表
- 审核报表
- 价格报表

以及它们下面的叶子节点。

#### 5. 根据当前选中的项目节点，拼出整套报表上下文

最关键的是：

- `buildReportWorkspaceContext()`

它会根据你现在点的是父节点还是造价文件节点，把：

- 左上已加载对象列表
- 左下报表树
- 右侧报表内容

整合成一个统一的数据结构，再交给报表工作区组件使用。

### 4.10 `src/views/cost/project-management/components/WorkbenchColumnNode.vue`

这个文件看起来很小，但非常省代码。

因为很多表格都不是普通单层表头，而是多层嵌套表头。

比如：

- 报审
  - 工程费用（元）
    - 小计
    - 设备购置费
    - 主要材料费

如果每个表格都手写，会非常啰嗦。

所以这个组件做的事情就是：

- 如果发现当前列下面还有子列，就继续递归渲染
- 如果没有子列，就渲染成普通列

你可以把它理解成：

- “把树形列配置自动变成真正表格列的转换器”

### 4.11 `src/views/cost/project-management/hooks/useOverlayPanelResize.ts`

这个文件专门处理右侧属性面板的拖拽行为。

可以把它理解成：

- “右侧面板的拖拽发动机”

它主要做了这些事情：

#### 1. 记录当前面板宽度

比如默认宽度是多少，拖动后变成多少。

#### 2. 支持收起

当拖动到足够窄时，它会自动收起。

#### 3. 支持恢复

收起后，再拖出来会恢复到上一次展开宽度。

#### 4. 统一管理鼠标事件

包括：

- `mousedown`
- `mousemove`
- `mouseup`

如果把这些逻辑全塞到页面组件里，会很乱。  
单独提出来之后，`index.vue` 会清爽很多。

## 6. 这些文件是怎么串起来的

如果按“用户点开页面”的顺序看，大概是这样的：

### 第一步：先通过路由进入页面

- `projectWorkbench.ts`

它把页面挂到：

- `/project-workbench/index`

### 第二步：Vue 应用得先认识 `vxe-table`

- `vxeTable.ts`
- `main.ts`

这一步做完后，后面的树表、复杂表格才都能正常用。

### 第三步：进入总控页面

- `index.vue`

这里决定：

- 现在处于哪个主页签
- 要显示哪套工作区
- 当前选中哪一行
- 要不要弹窗
- 要不要进报表

### 第四步：根据主页签，分别交给不同子组件

- `项目管理`：主要还是 `index.vue` 自己渲染
- `项目工料机汇总`：交给 `MaterialSummaryWorkspace.vue`
- `项目报表`：交给 `ProjectReportWorkspace.vue`

### 第五步：每个子页面再去用自己的 mock 数据

- 主界面用 `mock.ts`
- 工料机汇总用 `materialSummaryMock.ts`
- 项目报表用 `projectReportMock.ts`

### 第六步：所有数据结构都遵守 `types.ts`

这样不同文件之间才能对得上。

## 9. 一句话总结

这次新增的代码，不是只加了一个页面，而是搭了一整套“在线造价工作台”的基础骨架。

它已经把下面这些关键能力分开了：

- 页面入口
- 页面总控
- 工料机汇总子工作区
- 报表子工作区
- 主界面 mock 数据
- 报表 mock 数据
- 通用类型定义
- 面板拖拽逻辑

这样的好处是：

- 后面继续加功能，不容易全堆到一个文件里
- 新人也更容易知道“应该去哪个文件改”
- 从 mock 过渡到真实接口时，替换成本会低很多
