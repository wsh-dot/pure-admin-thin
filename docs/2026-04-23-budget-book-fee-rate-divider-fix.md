# 显示费率拖拽分隔条修复说明

## 本次修复内容

- 修复预算书工作区中 `取费程序 -> 显示费率` 场景下，费率明细表分隔条无法上下拖动的问题。
- 将费率明细区默认高度从 `300` 调整为 `180`。
- 将费率明细区最小高度从 `180` 调整为 `120`。
- 将上方取费程序树表的保底剩余高度从 `140` 调整为 `96`，避免上下区域的最小高度约束互相顶死。
- 给内部分隔条补充显式最小高度，提升可抓取性。

## 原因说明

- 原先 `显示费率` 双表布局里，上下两块区域的最小高度设置过大。
- 在当前详情区默认高度下，拖拽公式会把可用范围压缩成 0，表现出来就是分隔条看得到，但实际上拖不动。

## 影响文件

- `src/views/cost/project-management/budget-book.vue`
  - 调整费率明细区默认高度与最小高度配置。
  - 调整取费程序上半区最小保留高度。
  - 调整分隔条样式，增强拖拽命中区域。
